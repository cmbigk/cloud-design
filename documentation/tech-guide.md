# 📘 Technical Operations Guide

This guide provides the necessary commands and information to manage your cloud-design microservices project running on AWS.

---

## 🏗️ How This All Works

### 1. Key Terms to Know
*   **Terraform (The Architect):** A tool that writes text files to build real servers, networks, and databases on AWS. Instead of clicking buttons in the AWS console, we run code.
*   **EC2 (The Server):** Think of this as a virtual computer in an AWS data center. We are running a single `t3.small` instance to host everything.
*   **ECR (The Library):** **Elastic Container Registry**. This is where AWS stores your Docker images. After we build your app into a "container," we "push" (upload) it here so the server can "pull" (download) and run it.
*   **K3s (The Manager):** A lightweight version of Kubernetes. Its job is to make sure your 6 different apps (containers) are always running, talk to each other, and restart if they crash.
*   **VPC (The Private Office):** **Virtual Private Cloud**. A private network in AWS where your server lives. It keeps your databases hidden from the public internet for security.

### 2. The Step-by-Step Deployment Process
1.  **Infrastructure Creation:** We used **Terraform** to tell AWS: "Hey, I need a VPC, a server (EC2), and a library (ECR)."
2.  **Container Building:** We took your code in `src/` and used **Docker** to package it into "Images."
3.  **Library Upload (ECR):** We uploaded those images to **AWS ECR**.
4.  **Cluster Setup:** When the EC2 server started, it automatically installed **K3s**.
5.  **Running the Apps:** We used `kubectl` (the Kubernetes remote control) to tell K3s: "Go to the ECR library, pull our images, and run them as 6 services."

## 🖥️ How to Check Your Project in the AWS Console

Since we deployed to the **Stockholm (`eu-north-1`)** region, follow these steps:

### 1. Check your Server (EC2)
1.  Log in to the [AWS Management Console](https://console.aws.amazon.com/).
2.  In the top search bar, type **EC2** and click it.
3.  Make sure the region in the top-right corner is **Stockholm**.
4.  Click **Instances (running)**. You should see one instance named `k3s-single-node`.
5.  Check the **Public IPv4 address**—it should match `51.21.222.216`.

### 2. Check your Image Library (ECR)
1.  In the search bar, type **ECR**.
2.  Click **Elastic Container Registry**.
3.  Click **Repositories**. You will see 6 repositories (like `api-gateway`, `billing-app`, etc.).
4.  If you click on one, you can see the "Image tag" (latest) and when it was uploaded.

### 3. Check your Network (VPC)
1.  Search for **VPC**.
2.  Click **Your VPCs**. You will see `cloud-design-vpc`. This is the private "fence" around your project.

## 🛠️ Managing the Cluster Locally

To interact with Kubernetes from your local machine, use the `kubeconfig` generated by the `scripts/deploy.sh` script.

```bash
# Set your Kubeconfig environment variable so your computer knows where the server is
export KUBECONFIG=$(pwd)/kubeconfig/k3s.yaml

# Check if the server is healthy
kubectl get nodes

# See all 6 apps running!
kubectl get pods

# See the public access point (Ingress)
kubectl get ingress
```

## 🛑 HOW TO STOP EVERYTHING (To avoid costs)

To completely delete the AWS infrastructure and stop using your credits:
1.  Open your terminal.
2.  Navigate to the `terraform` folder: `cd terraform`
3.  Run the destroy command:
    ```bash
    terraform destroy -auto-approve
    ```
4.  **Wait** until it says "Destroy complete!" This will delete the EC2 instance, the VPC, and the ECR repositories so AWS stops billing you.

## 🛡️ Audit Defense (Bonus Answers)

- **Why use ECR?** ECR is private. Unlike Docker Hub (which is public), only our AWS server has permission to download these images.
- **Why use K3s instead of standard Kubernetes (EKS)?** EKS costs ~$72 per month just to exist. K3s is free and lightweight enough to run on our single free-tier server.
- **How is the DB protected?** The PostgreSQL databases are not given a public IP address. They "listen" only to connections from other apps inside the `cloud-design-vpc`.
